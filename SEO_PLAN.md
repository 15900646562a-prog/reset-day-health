# SEO 计划 · 实施时间表 · 验收标准

> COO 定。SEO 这条线的总规划。自治系统按此跑,COO 每日验收按此查,CEO 按此看进度。
> **一句话**:用免费自然搜索流量 → 落地页 → T-Patch 订单(北极星=泰国首单);建一个会复利、自治、据数据追赢家词的 SEO 资产。
> **诚实前提**:SEO 慢——收录要 1-2 周,有意义的转化要 8-12 周。**别用广告节奏(几天看 ROAS)判它失败。**

---

## 分阶段时间表 + 验收标准(每阶段达标=进下一阶段)

| 阶段 | 时间 | 干什么 | 🎯 验收标准(达标=过) |
|---|---|---|---|
| **P0 地基** ✅已完成 | 第 0 周 | 引擎自治 + 58 页双市场 + R14 口径 + 提交收录 | seo_qc 0 fail · 自动发布跑通 · sitemap 已交 · 每日自动验收已挂 |
| **P1 收录** | 第 1-2 周 | 等 Google 爬 + **接 GSC API** + 修收录问题 + 每周自动加页 | **≥70% 页被收录**(GSC 覆盖率)· `site:learn.resetday.health` 查得到 · GSC API 接通有数据 |
| **P2 曝光** | 第 2-6 周 | GSC 数据喂 ideate **追有曝光的词** + 修"有曝光没点击"(改标题/meta)+ 扩到 ~120 页 | **≥20 个词有曝光** · 出现首批自然点击 · 平均排名向上走 |
| **P3 点击→转化** | 第 6-12 周 | 自然流导 LP(UTM)+ 测哪个【簇×变体】转化 + 配合 LP 侧修"有点击没单" | **每周 ≥50 自然 session 到 LP** · **首笔 SEO 归因询单/订单** · 识别 ≥1 赢家【簇×变体】组合 |
| **P4 放大** | 第 3 月+ | 量产赢家簇的变体 + 加市场/簇 + 外链信号 | **可复现 SEO→单** · 知道单均获客成本 · 月稳定增长 |

## 总验收指标(COO 每日/每周看;接 GSC 后大部分自动)

| 维度 | 指标 | 数据源 | 目标(渐进) |
|---|---|---|---|
| **量** | 已发布 / 已收录页数 | 站点 / GSC | P1: 60→100 · P3: 150+ |
| **质** | seo_qc FAIL=0 · 口径 R14 · 抽审像真人 | seo_qc + COO 日审 | 持续 0 fail |
| **收录** | GSC 覆盖率 | GSC | P1 ≥70% |
| **曝光** | impressions / 出现词数 | GSC | P2 起量、月增 |
| **点击** | clicks / CTR / 平均排名 | GSC | P3 起量 |
| **转化** | SEO→LP session(UTM)· 归因订单 | UTM + LP `/api/stats` | P3 首单、识别赢家组合 |

> 红线指标(任一破=COO 立即处理/升级 CEO):seo_qc FAIL>0 · 出现违 R14 词(植物配方/等同 Mounjaro/编造疗效/无需处方)· ESCALATION 有新条目 · 自动发布连续失败。

## 关键依赖 / 风险(挡进度的)
1. **GSC API 接通** —— P2 起"数据驱动追赢家词"的前提。现卡 GCP 组织策略(禁服务账号密钥),待 CEO 走 OAuth 或开策略。**没接 = P2 靠盲扩,效率打折。**
2. **LP 侧订单回传** —— "哪个内容→单"的命门。LP 称已做(#3),**需真测一笔验证**。
3. **SEO 慢** —— 8-12 周才见转化,期间靠量+质+收录+曝光判进度,不靠订单。
4. **合规(R14/FDA)** —— seo_qc 自动守 + COO 日审复核;口径变更必须先进 `02_DECISIONS`。

## 分工
- **生产/发布/收录/扩量** = 自治系统(VPS cron + seo_qc 闸,无人值守)。
- **验收/考评/调方向/调上限/调口径** = COO(每日自动验收例程 + 周期调 `src/*`、`AUTONOMY_RULES`、本文件)。
- **一次性部署/凭据/重大口径决策** = CEO。

## 复盘节奏
- **每日**:COO 自动验收(量/质/收录/异常)→ log,P0 异常才提醒 CEO。
- **每周**:COO 看 GSC 趋势(接通后)→ 调 ideate 方向 / 上限 / 砍无量词。
- **每月**:对照本表阶段验收 → 决定进阶 / 放大 / 加市场;更新本文件。
