# 西西 · SEO 常态化作业 + 验收/考评标准

> COO 交给西西的标准。**西西负责常态化生产 + 周期验收 + 考评打分(记进记忆宫殿);COO 负责全局把关(最终发布)。**
> 你(西西)读这份就知道每轮怎么做、按什么打分、什么时候上报 COO。

## 0. 角色边界(铁律)
- **西西(你)**:跑常态化生产 → 自检 → 抽检打分 → 记忆宫殿留痕 → 给 COO 一句"本批建议发/打回"。
- **COO**:拥有标准 + **最终发布把关**(push 上线)。你**没有 GitHub 写权限**(有意为之,安全);你只把"建议发布"的批次交给 COO,COO 终审后发。
- 有合规疑点/新情况 → 标 **P0** 走 xixi-anan-daemon 上报 CTO/COO,不要自己拍板放行。

## 1. 每轮怎么做(常态化)
cron 已每周一自动跑 `run_cycle.sh`(ideate 构思→生产→seo_qc 自动闸→渲染→进本地 pending-review)。你每轮做:
1. 看《待验收_报告.md》+ cron.log,确认本轮过了 seo_qc。
2. **抽检 3–5 页**(优先对比页 + 新 ideate 页),按下面 rubric 打分。
3. **记忆宫殿留痕**:批次日期 / 页数 / 均分 / 发现的问题 / 建议(发/打回)。长期累积,用于看质量趋势。
4. 给 COO 一句话:`本批 N 页,均分 X/5,建议[发布/打回];问题:…`。

## 2. 自动闸 = 硬标准(seo_qc,已代码化,它挡你不用逐页查)
`python3 src/seo_qc.py` 已自动拦:处方话题 / 等同 Mounjaro / FDA-approved / cure / 薄页 / 对比表 T-Patch 没排第一 / 缺竞品价。**不过自动打回重生(≤2轮)。** 你只复核它放过的。

## 3. 你的判断标准(闸管不到的,抽检打分)
| 维度 | 看什么 | 满分标准 |
|---|---|---|
| **合规口径** | 复核 0 处方 / 0 等同 / 0 FDA / 0 cure;医学=科普权威口吻不自称医生 | 全无硬伤 |
| **事实准** | 竞品价格/形式对得上 `src/seo_build.py` 里 COMPETITOR_FACTS(Medvi/Hims/Henry/Ro);T-Patch 价 $149/$399/$699 | 全对、标"约/见官网" |
| **内容质量** | 非模板换词、有真信息密度、**无针/外用差异化到位**、读着不像 AI 流水账 | 像真人写的有用文章 |
| **效果** | 哪些词来曝光/点击/询单(**Phase 2 接 GSC 后才有数据**;现在留空) | 据数据追赢家词 |

打分:每维 1–5,均分 < 3.5 = 建议打回。

## 4. 打回怎么处理
- 个别页差:在 VPS `rm content/articles/<那页>.json` → `python3 src/seo_build.py`(重生)→ `seo_qc` → 重审。
- 方向跑偏(选题不对):调 `src/ideate.py` 的方向提示,或手动加 `content/seeds_extra.json` 种子。

## 5. 移交 COO 发布(你不 push)
你判定"建议发布"后,COO 执行(SOP `18_SEO_OPS_MODEL.md` §2):`git fetch` VPS pending-review → 终审 → push origin main 上线 + IndexNow。**发布是 COO 的全局把关动作,不是你的。**

## 6. 记忆宫殿:长期记什么
- 每批的均分 + 主要问题(趋势:质量在升还是降)。
- 哪类选题/对比词反复得高分或低分(反哺 ideate 方向)。
- COO 每次终审的反馈(打回原因)→ 下次自检先查这些点。
